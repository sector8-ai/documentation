Use this path when your application runs on Node.js or TypeScript and you want a direct SDK integration instead of calling the REST API yourself.

## Prerequisites

- Node.js 18 or later
- `SECTOR8_API_KEY`
- `SECTOR8_CLIENT_ID`

## Install the SDK

```bash
npm install @sector8/sdk
```

Source repository: `https://github.com/sector8-ai/sector8-sdk-typescript`

## Set your credentials

```bash
export SECTOR8_API_KEY=your_api_key
export SECTOR8_CLIENT_ID=your_client_id
```

## Send your first event

```typescript
import sector8 from '@sector8/sdk';

async function main() {
  const client = sector8.setup({
    apiKey: process.env.SECTOR8_API_KEY!,
    clientId: process.env.SECTOR8_CLIENT_ID ?? '1',
    enableAutoInstrumentation: false,
    debug: true,
  });

  const connected = await client.testConnection();
  console.log('Connected:', connected);

  await client.logLLMCall({
    provider: 'openai',
    model: 'gpt-4',
    tokens: 24,
    cost: 0.001,
    latency_ms: 120,
    success: true,
    prompt: 'Sector8 TypeScript SDK onboarding test',
    completion: 'Telemetry sent successfully',
  });

  await client.close();
}

main().catch(console.error);
```

Expected output:

```text
Connected: true
```

<Tip>
The TypeScript SDK defaults to the production Guard Module at `https://sdkapi.sector8.ai`. You only need `baseUrl` when targeting a non-production environment.
</Tip>

## Minimal setup

```typescript
import sector8 from '@sector8/sdk';

const client = sector8.setup({
  apiKey: process.env.SECTOR8_API_KEY!,
  clientId: process.env.SECTOR8_CLIENT_ID ?? '1',
});

await client.logLLMCall({
  provider: 'openai',
  model: 'gpt-4',
  tokens: 150,
  cost: 0.003,
});
```

## Verify the integration

1. Run:

```bash
curl https://sdkapi.sector8.ai/api/health
```

2. Confirm `client.testConnection()` returns `true`
3. Confirm the event appears in Sector8 telemetry or the dashboard

<Check>
You are done when `testConnection()` returns `true` and one `logLLMCall()` event is visible in Sector8.
</Check>

## Common mistakes

- Using the wrong host instead of `https://sdkapi.sector8.ai`
- Forgetting `SECTOR8_CLIENT_ID`
- assuming `testConnection()` proves logging works without sending a real event
